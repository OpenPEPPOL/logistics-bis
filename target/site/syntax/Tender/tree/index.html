Reference number
An identifier that is specified by the buyer and used as a reference number for all documents in the procurement process. It is also known as procurement project identifier, procurement reference number or contract folder identifier.

Tender issue date
The date when the tender is issued by the economic operator.

Tender issue time
The time when the tender is issued by the economic operator. The issue time must have granularity of seconds, and include time zone information

Example value: 09:00:00+01:00

1..n •   cac:DocumentReference

Provided document
Description of documents that are provided as part of a tender. Is used to describe the Tender, the call for tender reference and related documents like the ESPD response. Tender documents that are encrypted may point to the file extension .p7m in the description.

1..1 •   •   cbc:ID

Document identifier
A unique reference of the document reference. May point to the tender, call for tenders or related documents like the ESPD response. The cbc:ID of a Call for tenders reference creates a link between the call for tender and the tender.The call for tenders and tenders reference must be a UUID

1..1 •   •   cbc:DocumentTypeCode

Document type code
A code specifying the type of the document allowing to distinguish between different document types. Allowed values are 310 (Offer/quotation) for Tender, 311 (Request for quote) for Call for tender reference and 916 (Related document) for ESPD response

M •   •   •   @listID

Fixed value: UNCL1001
